раздел первый

архитектура: django + next.js?

вы выбрали и Django, и Next.js — это две разные технологии, нужно определиться

противоречие из раунда 1 В поле «устройство» вы написали «хочу сделать на Python Django». Но также выбрали Next.js как фронтенд. Django — это Python-фреймворк, Next.js — JavaScript-фреймворк. Их можно использовать вместе, но это усложняет проект. Нужно выбрать одну из трёх схем.
выберите, как будет устроено приложение внутри

Монолит на Django — один Python-проект. Django сам рисует страницы через шаблоны (HTML генерируется на сервере). Проще запустить, проще деплоить. Минус: интерфейс чуть менее «отзывчивый», чем у современных SPA.

Django API + Next.js — два проекта. Django отдаёт данные через JSON (REST API), Next.js рисует интерфейс в браузере. Современный подход, хорошая UX. Минус: нужно поднимать и деплоить два приложения.

Только Next.js + Supabase — вообще без Python. Next.js делает всё: и интерфейс, и логику. Supabase — база данных с готовой авторизацией. Самый быстрый старт, но Python нет совсем.

раздел второй

база данных: sqlite не подходит для публичного сервиса

вы выбрали публичный сервис и sqlite — это проблема

проблема из раунда 1 SQLite хранит данные в одном файле. Он не умеет работать с несколькими пользователями одновременно — при двух одновременных записях данные могут испортиться. Для публичного сервиса нужна PostgreSQL.
