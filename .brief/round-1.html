технологии

выберите технологический стек — или доверьтесь клоду

как пользователи будут входить в приложение?

Авторизация — это «замок» на двери приложения. Без авторизации любой может открыть ваше приложение и увидеть чужие данные.

Email + пароль — классика. Пользователь придумывает пароль. Магическая ссылка — он вводит email, получает письмо со ссылкой, кликает — и сразу внутри (без пароля). Google / GitHub — кнопка «Войти через Google».

на чём написать интерфейс приложения?

Это инструмент, на котором написан визуальный интерфейс — кнопки, формы, страницы. Если у вас нет предпочтений, выберите рекомендуемое.

Next.js — самый популярный выбор для веба сегодня. React + Vite — чуть проще, без серверного рендеринга. Vue — хорошая альтернатива React.

где хранить данные пользователей — расходы, категории, настройки?

База данных — это «шкаф с ящиками», где хранятся все данные. Расходы, пользователи, категории — всё там.

Supabase — Postgres в облаке с бесплатным тарифом и встроенной авторизацией. PostgreSQL — самостоятельный сервер. SQLite — файл на диске, подходит для простых/локальных проектов.

где будет жить готовое приложение?

Хостинг — это сервер, на котором работает ваш сайт. Когда кто-то открывает ваш адрес в браузере, запрос приходит на хостинг.

Vercel — бесплатно для небольших проектов, один клик из GitHub. VPS — арендуете сервер, полный контроль. Локально — только на вашем компьютере, не доступно другим.
