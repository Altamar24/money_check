# Money Check — техническая спецификация

## Обзор

Money Check — публичный веб-сервис учёта личных расходов. Целевой пользователь — человек, который хочет понять, куда уходят деньги, и контролировать месячный бюджет. Он вносит расходы с телефона сразу после каждой покупки, раз в неделю смотрит статистику по категориям. Приложение работает в браузере (мобильный + десктоп) и не требует установки.

---

## Пользовательские истории

- Как новый пользователь, я хочу зарегистрироваться по email и паролю, чтобы мои данные хранились отдельно от других.
- Как пользователь стоя на кассе, я хочу за 5 секунд добавить расход (сумма → категория → готово), чтобы не задерживать очередь.
- Как пользователь, я хочу видеть, сколько уже потратил из месячного бюджета, чтобы не выйти за рамки.
- Как пользователь, я хочу видеть круговую диаграмму расходов по категориям за месяц, чтобы понять, на что уходит больше всего.
- Как пользователь, я хочу добавить повторяющийся платёж (аренда, подписки), чтобы не вносить его каждый месяц вручную.
- Как пользователь, я хочу выгрузить все транзакции в CSV и открыть в Excel.

## Функции v0.1

### 1. Авторизация
Регистрация по email + пароль, вход, выход, сброс пароля по email.

**Критерии приёмки:**
- Пользователь может зарегистрироваться; при дублировании email — понятная ошибка.
- После входа сессия живёт 30 дней (remember me по умолчанию).
- Данные одного пользователя недоступны другому.

---

### 2. Быстрое добавление транзакции
Форма добавления расхода или дохода оптимизирована для телефона.

**Поля:**
- Тип: расход / доход
- Сумма (обязательно) — числовая клавиатура
- Категория (обязательно) — сначала показываются последние 4 использованные
- Дата (обязательно) — по умолчанию сегодня
- Заметка (необязательно)
- Повторяющийся (чекбокс) — если включено: интервал (ежемесячно / еженедельно)

**Критерии приёмки:**
- Кнопка «+» видна на главном экране без прокрутки.
- Форма открывается в 1 тап, заполнение занимает не более 3 полей для простого расхода.
- После сохранения пользователь возвращается на предыдущий экран; в шапке мигает подтверждение.
- На мобильном поле «сумма» открывает числовую (не буквенную) клавиатуру.

---

### 3. Категории
Система предустановленных категорий + возможность создать свои.

**Встроенные категории:**

| Название | Иконка |
|---|---|
| Еда и рестораны | 🍔 |
| Продукты | 🛒 |
| Транспорт | 🚌 |
| Топливо | ⛽ |
| Жильё / аренда | 🏠 |
| Коммунальные | 💡 |
| Здоровье | 💊 |
| Одежда | 👕 |
| Развлечения | 🎬 |
| Подписки | 📱 |
| Образование | 📚 |
| Прочее | 📦 |
| Зарплата _(доход)_ | 💼 |
| Прочий доход _(доход)_ | 💰 |

**Критерии приёмки:**
- Пользователь может создать категорию с именем и выбором цвета (8 вариантов).
- Пользователь может переименовать или скрыть свою категорию, но не удалить системную.
- Скрытые категории не появляются в форме добавления.

---

### 4. Месячный бюджет
Один общий лимит расходов на текущий месяц.

**Критерии приёмки:**
- Пользователь устанавливает числовой лимит на месяц в настройках.
- На главном экране видны: потрачено / лимит / остаток.
- Прогресс-бар меняет цвет: зелёный (< 70%), жёлтый (70–90%), красный (> 90%).
- Лимит применяется только к расходам, доходы не учитываются.
- Если бюджет не задан — блок скрыт.

---

### 5. История транзакций с фильтрами
Список всех транзакций с возможностью фильтрации и поиска.

**Фильтры:**
- Период: текущий месяц / прошлый месяц / произвольный диапазон дат
- Тип: все / только расходы / только доходы
- Категория: одна или несколько

**Критерии приёмки:**
- По умолчанию показывается текущий месяц, сортировка по дате (новые сверху).
- Каждая запись: дата, категория (иконка + название), заметка, сумма.
- Тап на запись открывает редактирование или удаление.
- Итог по текущему фильтру показан в шапке списка.

---

### 6. Статистика
Аналитика расходов за выбранный период.

**Содержимое страницы:**
- Переключатель периода: текущий месяц / прошлый месяц
- **Круговая диаграмма** расходов по категориям _(выбор claude — наглядно показывает, куда уходит больше всего)_
- **Прогресс-бар бюджета** (потрачено / лимит) _(выбор claude — главный мотиватор не превышать лимит)_
- Топ-3 категории по сумме расходов
- Итог: расходы / доходы / баланс за период

**Критерии приёмки:**
- Диаграмма строится на Chart.js (CDN), без отдельного пакета.
- При клике на сегмент диаграммы — переход в список транзакций с этой категорией.
- Если расходов нет — показывается заглушка «пока нет данных».

---

### 7. Повторяющиеся платежи
Автоматическое добавление транзакции по расписанию.

**Критерии приёмки:**
- Пользователь отмечает транзакцию как повторяющуюся и выбирает интервал: ежемесячно (конкретная дата) или еженедельно (конкретный день недели).
- Django management command `create_recurring` добавляет все просроченные повторения при запуске; запускается через cron раз в сутки.
- Пользователь видит список своих повторяющихся платежей и может их отключить.

---

### 8. Экспорт CSV
Выгрузка всех транзакций за выбранный период.

**Формат файла:** `moneycheck_YYYY-MM.csv`

| Поле | Пример |
|---|---|
| date | 2026-04-15 |
| type | expense |
| amount | 1500.00 |
| category | Еда и рестораны |
| note | Обед с коллегами |

**Критерии приёмки:**
- Кнопка «Экспорт CSV» доступна на странице истории.
- Экспортируются транзакции текущего активного фильтра (период + категории).
- Разделитель — запятая, кодировка UTF-8 с BOM (для корректного открытия в Excel).

## UI / UX

Минималистичный интерфейс с упором на мобильный браузер. Одна главная кнопка «+ Добавить» — всегда видна без прокрутки. Форма добавления занимает максимум 3 обязательных поля. Интерфейс работает на экранах от 320px. Тёмная тема не обязательна в v0.1.

Цветовая схема: нейтральный фон, синий акцент (`#2563eb`), красный для перерасхода (`#dc2626`), зелёный для дохода / положительного баланса (`#16a34a`).

## Нефункциональные требования

- Страница загружается за < 2 секунды на мобильном 4G.
- Форма добавления транзакции работает без JavaScript (базовый Django form submit), HTMX — прогрессивное улучшение.
- Все данные пользователя изолированы через `request.user` фильтрацию на уровне queryset.
- Пароли хранятся в виде bcrypt-хешей (Django default).
- CSRF-защита включена на всех формах.
- Резервное копирование PostgreSQL — pg_dump в cron раз в сутки.

## Вне scope v0.1

| Функция | Причина |
|---|---|
| Нативное мобильное приложение | требует отдельного деплоя и публикации в сторах; PWA достаточно |
| Импорт банковской выписки | нет стандарта между банками, требует парсеры — отложить после запуска |
| Учёт инвестиций / активов | другая предметная область — отдельный продукт |
| Push-уведомления | требует service worker + VAPID ключи; добавить в v0.2 |
| AI-анализ расходов | требует API ключи и стоимость — только после проверки спроса |
| Telegram-бот | дублирует веб-интерфейс; рассмотреть после запуска |
| Бюджет по категориям | усложняет UX; общего бюджета достаточно для v0.1 |
| Несколько валют | усложняет модель данных; добавить если пользователи попросят |

## Открытые вопросы

- **Импорт банковской выписки** — пользователь не определился. Рекомендация: отложить до v0.2, оценить спрос после запуска.
- **Часовые пояса** — приложение публичное, пользователи могут быть из разных часовых поясов. В v0.1 хранить даты как `date` (без времени) и принимать серверное время. Если появятся жалобы — добавить `timezone` в профиль.
